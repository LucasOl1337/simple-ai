# Screenshots

## Objetivo

Guardar capturas curatoriais de referencias visuais para consulta interna por humanos e agentes.

## Politica

- Uso interno no projeto.
- Toda captura deve ter URL de origem registrada em `agent-ready/screenshot-index.json`.
- Sempre adicionar nota curta explicando o que vale aprender com a captura.
- Nao usar capturas como ativos finais do produto.

## Nomeacao sugerida

- `segment-style-layout-01.png`
- exemplos:
  - `clinic-clean-trust-01.png`
  - `bakery-warm-hero-01.png`
  - `catalog-grid-scannable-01.png`
